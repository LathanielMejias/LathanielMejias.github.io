Bilingual Ebook/Audiobook Reader

Technologies
Python NodeJS Javascript HTML CSS
Gallery
Project Description

I wanted an easy way to read books in foreign languages, hear the audio (for correct pronunciation), and save useful vocabulary.

I started with the epub file and the mp3 audiobook file and added the following:
  • Synchronized audio with text at the sentence level
    • Clicking any sentence seeks to its corresponding segment in the audio
    • Seeking anywhere in the audio will take you to the corresponding text
    • Automatically highlights text as the audio plays
  • Can navigate by sentence on the bottom left
    • Previous, Replay, Next
  • Added pagination
    • Split the epub into about 90 "pages"
    • Allows you to bookmark specific pages and keep your place
    • Page navigation gets added to browser history, so you can press Back to get to previous pages
    • Your place in the audio/text will be remembered after page refreshes
  • Added a search function that works across pages
    • Supports Match Case and Whole Word searches
    • Can select the word and press Ctrl+F to open the search or click the magnifying glass button
  • Matched Italian sentences to English paragraphs
    • Can click on the bar at the bottom to open the corresponding English section
  • Added an Anki (flashcard program) button that appears on every sentence
    • Adds the sentence and audio to the note automatically
    • English translation can be manually copied from the bottom English panel
    • Contains basic WYSIWYG to style Anki fields
    • Sentences saved to Anki will be subtly underlined afterwards (which persists after refreshing)

Kodansha's Kanji Learning Course Android App

Technologies
Ionic Javascript HTML CSS
Gallery
Project Description

Background: The Japanese language has 2,136 official characters called "Kanji" that are expected to be learned to be able to consume everyday content. Not only that, but the pronunciation ("reading") of each character can differ completely depending on which word it appears in. So, to aid in my learning Japanese, I got a book called "The Kodansha Kanji Learner's Course," and while it is very thorough and helpful, it is quite a hefty book to lug around. But, my cell phone, however, is not! So, I created a web app that can compile into an Android app with the help of the Ionic Framework.

With the app, I can:

  • Navigate through all 2,136 Kanji and some relevant vocabulary words
  • View its stroke order (which is standardized)
  • Hide/Show the Kanji's Readings and English meaning
  • Tap any other Kanji in a word to cross-reference its entry
  • Navigate back through the history of visited Kanji entries

Automated Coffee Scale

Technologies
Raspberry Pi Vue.js Python Javascript HTML CSS
Gallery
Project Description

Since I like to try different coffee beans, I make sure to weigh out my beans each time I make coffee to standardize the tasting process. This got a little tedious doing it every day, sometimes 2-3 times a day. I know they make grinders with built-in scales, but my grinder was fairly new and working well, so I decided to create my own scale that could make use of my existing grinder.

I got a Raspberry Pi, a weight sensor, a touchscreen and some legos, and got to work. I decided to go with a web app, this way I could access the site from my phone if I ever felt the need to start grinding my coffee remotely. I wanted some quick and simple, so I used Vue.JS here for the frontend and Python for the server and hardware control. The display automatically turns on after touching anywhere on it, and shuts off after inactivity.

The site allows me to:

  • Select the ratio of coffee to water
  • Set how much coffee to grind based on different units (e.g. ounces, milliliters, cups, and servings)
  • Manually start and stop the grinder (since the hardware switch is now always switched on)

Technologies
Raspberry Pi Python
Gallery
Project Description

My previous employer used to ring this very bell manually whenever a big sale went through. Someone wondered if we could automate something, and I came up with this. I used a Raspberry Pi Zero W to monitor Salesforce for sales, and when it detects one, it drives the solenoid which rings the bell. Smaller sales would ding once while bigger sales would ding 3 times.
